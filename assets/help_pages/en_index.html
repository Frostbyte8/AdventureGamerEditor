Adventure Gamer Editor++

Adventure Gamer Editor++ is a map editor for Adventure Gamer for Windows.

Requirements

  • Windows
  • Web Browser that has at least HTML4 support. (Required for Help)

Features

  • Player can start anywhere on the Map, and not just the Start Feature
  • Dark Tiles can be made without corresponding switches
  • Jump pads can be made to jump to any tile
  • True One-Way paths by not imposing which roads connect to each other
  • Create worlds as small as 3x3, and as large as 100x100
  • Edit the Gates of Macrohard and see all the text not visible during regular play
  • Unicode file paths are supported (but not file names!)

The editor supports the following operating systems:

  • Windows 2000/XP*
  • Windows Vista
  • Windows 7
  • Windows 8 / Windows 8.1
  • Windows 10

*Windows 2000 and XP support is a work in progress, and as such, may not work perfectly.

The program may run on newer versions of Windows, but those are untested.

Map Editor

This is the main editor. On the left is the road selector palette, it contains all 32 road types. There is no need to switch between grass and dirt tiles anymore.

The center pane is the actual map. You can draw tiles here by double clicking, or pressing space. You can also move the selection box around with the arrow keys. Right click will open the tile submenu on the currently selected tile.

The bottom pane contains information about what objects and characters are on the currently selected tile.

Finally, the right most side contains the list of objects and characters that can placed and edited, as well as buttons to add and delete characters.

You can edit the world's summary and story from the "World" menu, as well as the player's attributes, the name of the world, and the name of the currency.

If you find the map too small you can zoom in with the zoom menu, up to 4 times the original size.

Adding/Editing Objects

In this dialog you can add, or edit existing objects. Note that because Adventure Gamer for Windows is a 16-bit application, you are limited to the characters that Windows 3.1 could accept. This means that you cannot use Unicode strings here.
